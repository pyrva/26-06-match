More than a match

The magic that is structural pattern matching in Python

PyRVA • June 2026

Three-minute setup before we go hands-on. I'll show what switch actually does in other languages, and why Python's match is a different thing entirely — not a switch.

Every language has this


if command == "quit":
    ...
elif command == "look":
    ...
elif command == "go":
    ...
else:
    ...

We keep typing command == … command == … command ==

Comparing ONE value against many possibilities.

Every language on Earth has this. It works. But notice we keep typing command == over and over. We're comparing one value against a bunch of possibilities. Any time you catch yourself writing that ladder, most languages hand you a tool to tidy it up. In C, Java, JavaScript — switch.

JavaScript: tidy it up with switch


switch (command) {
  case "quit":
    console.log("Goodbye");
    break;
  case "look":
    console.log("You see a key.");
    break;
  default:
    console.log("Huh?");
}

Cleaner — but there are two things switch does that differ from Python.

Same idea in JavaScript — a switch. Cleaner. But there are two things switch does that matter for tonight, and you've probably hit both.

"Multiple matches" — stack the labels


switch (day) {
  case "Mon":
  case "Tue":
  case "Wed":
  case "Thu":
  case "Fri":
    console.log("Weekday");   // all five reach here
    break;
  case "Sat":
  case "Sun":
    console.log("Weekend");
    break;
}

Several cases → one block.

If five days are all weekdays, you just stack the labels. That's the "multiple matches" idea — several cases feeding one block.

The famous footgun — forget the break


switch (command) {
  case "quit":
    console.log("Goodbye");
    // no break — oops
  case "look":
    console.log("You see a key.");
    break;
}

command === "quit" prints BOTH lines — it keeps going until a break.

One value, two case bodies. That's "do more than one thing."

Now watch this. Forget the break. command === quit prints both Goodbye AND You see a key. It just keeps going. So a switch can literally do more than one thing for a single value — it runs through every case until it hits a break. That's flexible, and it's the most-forgotten keyword in the C family. Half of all switch bugs are missing breaks.

Every case is just === in a costume


switch (command) {
  case "quit":
    console.log("Goodbye");
    break;
  case "look":
    console.log("You see a key.");
    break;
  default:
    console.log("Huh?");
}

Switch matches on value equality — only value equality.

It cannot ask: "is this a list that starts with 'go'?"

Here's the key thing. Look at what every case is actually doing. case quit just asks "is command strictly equal to quit?" — it's an equality check in a costume. Switch matches on value equality, and only value equality. It can't ask "is this a two-element list that starts with go?" — only "is it exactly this value?"

Now Python — same idea, new word: match


match command:
    case "quit":
        return "Goodbye!"
    case "go":
        return "You move."
    case _:
        return "Huh?"

Three differences. Each one breaks a switch assumption.

Now Python. Same idea, different word — match. Three differences. Each one shatters a switch assumption.

Break #1 — no fall-through. No break. Period.


match command:
    case "quit":
        ...            # first match runs, then STOP
    case "look":       # ← reached ONLY if "quit" didn't match
        ...

That JS footgun is impossible in Python — one value can never fire two case bodies.

First — there is no break, because there is no fall-through. The first case that matches runs, and then you're done. That JS footgun from a minute ago is impossible in Python — one value can never trigger two case blocks.

Break #2 — "multiple matches" uses |


match day:
    case "Mon" | "Tue" | "Wed" | "Thu" | "Fri":
        return "Weekday"
    case "Sat" | "Sun":
        return "Weekend"

One line. Explicit. No empty stacked labels.

Remember the stacked weekdays? In Python you spell it with a pipe. One line. Explicit. No empty case labels stacked on top of each other.

Break #3 — the real difference: structure, not equality


match command:
    case ["go", direction]:
        return f"You go {direction}."

This is not command == ["go", direction].

It asks: "two-piece sequence whose first piece is 'go'? Hand me the second."

["go", "north"] → direction = "north"

Switch can't do this. At all.

Third — and this is the whole point of tonight. Watch this one line. That case is NOT asking "is command exactly equal to this list?" It's asking "is this a two-piece sequence whose first piece is the word go? If so, hand me the second piece and call it direction." Run it with go north — direction becomes north. Switch can't do this. There's no version of it. It's a different operation: it takes data apart.

It's this, generalized


first, *rest = [1, 2, 3]   # first = 1, rest = [2, 3]

A switch is === with extra breaks to babysit.

match inspects the shape of your data and pulls it apart.

Destructuring, not switching.

destructuring - match inspects the shape of your data and pulls it apart. Tonight, that one idea —, not switching — explains every single pattern we use.
