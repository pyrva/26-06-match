    case "quit":

    case "look":

    case _:

3OR Patterns & Guards

match words:
    case ["go", ("north" | "n") as direction]:
        move(direction)
    case ["go", direction] if direction in player.current_room.exits:
        move(direction)
    case ["go", _]:
        return "You can't go that way."
  1. ["go", "n"] — what does direction hold?
  2. ["go", "west"] when there’s no west exit — what happens?
  3. ["go", "up"] — what happens?
uv run pytest tests/test_level3_or_guards.py -v

4Class Patterns

match event:
    case Click((x, y)):
        print(f"Clicked at {x}, {y}")
    case KeyPress("q") | KeyPress("Q"):
        print("Quit!")
    case KeyPress(key) if key in "wasd":
        print(f"Move: {key}")
  1. Click((3, 7)) — what are x and y?
  2. Does KeyPress("q") match the OR pattern?
  3. Does KeyPress("w") match the guarded case?
  4. What does the case for Quit() look like?
uv run pytest tests/test_level4_class_patterns.py -v

?Pulse check · order matters

match KeyPress("w"):
    case KeyPress("q") | KeyPress("Q"):
        print("quit")
    case KeyPress(key) if key in "wasd":
        print(f"move {key}")
    case KeyPress(key):
        print(f"unknown: {key}")
  1. What prints?
One question answers everything: “What would unpacking do here?”   Variables are assignment targets, not comparisons. _ is the wildcard. First match wins.
